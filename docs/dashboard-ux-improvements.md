## 2. Текущая дизайн-система

### Подход к стилизации

- **SCSS** (не CSS Modules, не styled-components)
- БЭМ-подобные имена: `ecos-dashboard__header`, `dashlet__body-content`
- Глобальные константы в `src/styles/constants.scss`
- CSS custom properties в `src/styles/variables.scss`
- Переиспользуемые паттерны в `src/styles/content.scss`

### Цветовая палитра

**Основные:**
- `$primary: #4b75b7` (тёмно-синий)
- `--primary-color: #4b75b7`
- `--dark-primary-color: #7396cd`
- `--light-primary-color: #c6d6f0`

**Синие оттенки:** `#eef0f8`, `#4b75b7`, `#2a446c`, `#0075ff`, `#7396cd`, `#c6d6f0`, `#f8fbfd`, `#f3f7f9`, `#b2d5ff`

**Серые оттенки:** 27+ оттенков от `#141414` (тёмный) до `#f8f8f8` (светлый)

**Статусные:** `$success: #00a707`, `$danger: #ec4f1d`, `$warning: #ffc700`

**Текст:** `$text-color-dark: #444444`, `$text-color-gray: #767676`

**Фон страницы:** `$page-background-color: #f8f9fa`

### Типографика

- Шрифт: `'Open Sans', sans-serif`
- Размеры: 36/24/20/16/15/14/12/11px
- Начертания: 400 (normal), 600 (semi-bold), 700 (bold)

### Отступы и раскладка

- Высота инпута формы: 40px
- Стандартные промежутки: 8px, 16px (flex-обёртки)
- Высота строки таблицы: 40px, заголовка: 44px
- Отступы заголовка дашборда: 0 18px
- Отступ контента Dashlet: 20px сверху

### Скругления углов (border-radius)

- Маленький: 4px
- Стандартный: 6px
- Большой: 8px
- Увеличенный: 12px
- Максимальный: 100px

### Тени

- По умолчанию: `rgba(0,0,0,0.16) 0px 1px 4px`
- При наведении: `0px 2px 9px rgba(0,0,0,0.15), 0px 6px 8px rgba(0,0,0,0.1), 0px 4px 8px rgba(0,0,0,0.1)`

### Стили ключевых компонентов

- **Контейнер Dashlet:** border-radius 12px, заголовок 48px, border 1px solid gray-4
- **Панель:** border 1px, padding-top 8px, заголовок 60px
- **Модальное окно:** border-radius 12px, заголовок 58px, padding body 20px
- **Таблица/Grid:** строка 40px, заголовок 44px, hover border -> brown-2
- **Кнопки:** высота 40px (стандарт), 26px (узкий), border-radius 8px

## 3. Текущее состояние — что работает хорошо

1. **Консистентная цветовая палитра** — синий primary (#4b75b7), серые для фонов
2. **Чёткая визуальная иерархия** — заголовок проекта -> табы -> виджеты
3. **Функциональная 2-колоночная раскладка** — основной контент + сайдбар для case-details
4. **Хорошая система скруглений** — 12px для контейнеров, 6px для инпутов, 8px для кнопок
5. **Зрелая система дизайн-токенов** — хорошо организованные SCSS-константы

## 4. Проблемы и предложения по улучшению

### 4.1 Контейнеры виджетов (Dashlet) — визуальное разделение

**Проблема:** Виджеты «Metadata», «Status», «Actions», «Stages» визуально сливаются друг с другом. Тонкая серая рамка (`1px solid $color-gray-4`) недостаточно их разделяет.

**Предложение:**
- Заменить border на мягкую тень: `rgba(0,0,0,0.06) 0 1px 3px, rgba(0,0,0,0.04) 0 1px 2px` (современный подход, аналогично Google Workspace, Notion)
- Немного затемнить фон страницы до `#f4f5f7` или `#f0f2f5` для контраста с белыми виджетами

### 4.2 Заголовки виджетов (Dashlet Header) — информационная перегрузка

**Проблема:** Заголовок виджета «Metadata» содержит 4 icon-only кнопки (редактирование, обновление, настройки, вид) без подписей и тултипов. Нарушает правила `aria-labels` и `nav-label-icon`. Пользователю сложно понять назначение каждой иконки.

**Предложение:**
- Показывать action-кнопки **при наведении** на заголовок виджета, а не постоянно — уменьшает визуальный шум
- Добавить `title`/`aria-label` к каждой кнопке
- Для кнопки сворачивания — улучшить анимацию `ease-out 200ms`

### 4.3 Виджет «Actions» — плоский список без приоритизации

**Проблема:** «Delete», «Edit Properties», «Change status», «Move» отображаются одинаково. Деструктивное действие «Delete» (красный текст) стоит первым — это anti-pattern по правилу `destructive-nav-separation`.

**Предложение:**
- Переместить «Delete» в конец списка или в overflow-меню (...)
- Сделать основные действия (Edit Properties, Change status) более заметными — стиль primary-кнопки
- Добавить иконки к действиям для быстрого распознавания
- Рассмотреть горизонтальную раскладку для 3-4 действий вместо вертикального списка

### 4.4 Виджет «Status» — неэффективное использование пространства

**Проблема:** Виджет Status занимает всю ширину правой колонки, но содержит только текст «In progress» с бейджем. Много пустого пространства.

**Предложение:**
- Объединить Status + Actions в один виджет сайдбара — заголовок «Status: In progress», ниже действия
- Или вынести Status в заголовок дашборда рядом с «Стройка» — экономит вертикальное пространство
- Добавить цветовой индикатор (progress bar или визуализация таймлайна)

### 4.5 Табы формы Metadata — навигация внутри виджета

**Проблема:** Табы «Main, Risks, Associations, Deadlines, Budget, Team» занимают горизонтальное пространство. На узких экранах могут не помещаться. «Табы внутри табов» (табы дашборда -> табы формы) могут путать пользователя.

**Предложение:**
- Использовать вертикальные табы/сайдбар-навигацию внутри формы при ширине > 900px
- Или заменить на pill-style табы с горизонтальным скроллом для мобильных
- Визуально дифференцировать табы дашборда (General/Tab/History/Documents) от табов формы — другой стиль

### 4.6 Таблица «Stages» — заголовки колонок и тулбар

**Проблема:**
- Тулбар таблицы (+ шестерёнка Actions_over_1 скачать) — иконки без подписей
- «Actions over 1» — непонятная формулировка для пользователя
- Длинные заголовки колонок («Planned completion date», «Planned income») обрезаются

**Предложение:**
- Добавить тултипы ко всем icon-only кнопкам в тулбаре
- Переименовать «Actions over 1» -> «Массовые действия (1)» или показывать только при выбранных записях
- Для длинных заголовков — разрешить перенос строки (wrap) или использовать сокращения с тултипом

### 4.7 Хлебные крошки и навигация

**Проблема:** Единственная хлебная крошка «Стройка» не показывает полный путь. Нет визуального понимания, откуда пришёл пользователь.

**Предложение:**
- Показать полный путь: «Проекты -> Стройка» — помогает ориентироваться (правило `breadcrumb-web` для иерархий 3+ уровней)
- Сделать хлебные крошки кликабельными для навигации назад

### 4.8 Сайдбар (левая панель) — визуальная иерархия

**Проблема:** Секции «Tasks» и «Sections» визуально слабо разделены. Бейдж «31» на Active tasks и «0» на Subordinates tasks — «0» не несёт информации и добавляет визуальный шум.

**Предложение:**
- Скрывать бейдж при значении 0 — показывать только когда есть содержимое (правило `tab-badge`)
- Добавить разделитель или больший отступ между секциями Tasks и Sections
- Рассмотреть сворачиваемые секции в сайдбаре

### 4.9 Общие улучшения — современные тенденции

| Область | Текущее | Предложение |
|---------|---------|-------------|
| **Фон страницы** | `#f8f9fa` (почти белый) | `#f4f5f7` или `#f0f2f5` для контраста с белыми виджетами |
| **Тени** | `rgba(0,0,0,0.16) 0 1px 4px` | Мягче: `rgba(0,0,0,0.08) 0 1px 3px` — тренд на деликатные тени |
| **Skeleton loading** | Не виден при загрузке | Skeleton loading для каждого виджета (правило `progressive-loading`) |
| **Пустые состояния** | Не проверены | Дружелюбные empty states с иконкой + описанием + CTA |
| **Drag & drop** | Поддерживается | Показывать drag handle только при наведении |

### 4.10 Доступность (критично)

- Icon-only кнопки в заголовках виджетов без `aria-label` — нарушает WCAG
- Стрелка сворачивания/разворачивания — нужен атрибут `aria-expanded`
- Фокусное кольцо — проверить видимость при Tab-навигации
- Контраст бейджа «0» (серый на белом) — может не проходить 4.5:1

## 7. Использованные UX-правила

Из гайдлайнов UI/UX Pro Max:

- `aria-labels` (Приоритет 1: Доступность) — aria-label для icon-only кнопок
- `destructive-nav-separation` (Приоритет 9: Навигация) — деструктивные действия визуально отделены
- `tab-badge` (Приоритет 9: Навигация) — бейджи на элементах навигации экономно
- `breadcrumb-web` (Приоритет 9: Навигация) — хлебные крошки для иерархий 3+ уровней
- `progressive-loading` (Приоритет 3: Производительность) — skeleton screens вместо спиннеров
- `nav-label-icon` (Приоритет 9: Навигация) — элементы должны иметь и иконку, и подпись
- `primary-action` (Приоритет 4: Стиль) — одна primary CTA на экран
- `whitespace-balance` (Приоритет 6: Типографика) — whitespace для группировки/разделения
- `empty-states` (Приоритет 8: Формы) — понятное сообщение и действие при отсутствии контента
- `focus-states` (Приоритет 1: Доступность) — видимые focus rings 2-4px
